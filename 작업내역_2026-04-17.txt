1. 흰 화면 버그 원인 분석 및 수정 (localStorage QuotaExceededError)
   ──────────────────────────────────────────────────────────────────
   증상:
   - 비관리자 계정에서 국내금(goldKr) 히스토리 데이터를 업로드하면
     화면이 하얗게 변하고 아무것도 표시되지 않음
   - 관리자 계정에서는 동일한 업로드가 정상 동작

   원인:
   - 브라우저 콘솔 에러:
     QuotaExceededError: Failed to execute 'setItem' on 'Storage':
     Setting the value of 'portfolioState_v5_gen2025ag@gmail.com' exceeded the quota.
   - 국내금 히스토리(수백~수천 건)를 포함한 전체 state를 localStorage에 저장 시도
   - 브라우저 localStorage 한도(~5MB) 초과 → try-catch 없이 에러 발생
   - React 렌더링 전체 크래시 → 흰 화면

   수정 내용 (src/App.tsx):
   - localStorage.setItem 호출을 try-catch로 감쌈
   - 1차 실패 시: indicatorHistoryMap, stockHistoryMap 제외한 경량 버전으로 재저장 시도
   - 2차 실패 시: 에러 무시 (Drive 백업에 의존), 앱 크래시 방지


2. 시장지표 히스토리 자동 수집 기능 추가
   ────────────────────────────────────────
   문제:
   - 국내금(goldKr)은 driveStatus='saved' 이후 자동 크롤링 로직이 있었음
   - 그러나 stooq 지원 지표 5개(us10y, goldIntl, usdkrw, dxy, vix)는
     히스토리 데이터가 없거나 오래되어도 자동 수집 없음
   - 사용자가 수동으로 📥 버튼을 클릭해야만 그래프에 반영됨

   수정 내용 (src/App.tsx):
   - stooqAutoCrawledRef 추가 (세션 당 1회 실행 보장)
   - 새 useEffect 추가:
     · driveStatus='saved' 완료 후 실행
     · us10y, goldIntl, usdkrw, dxy, vix 각각 데이터 상태 점검
     · 히스토리 없음 OR 최신 데이터가 3일 이상 지난 경우 → stooq에서 자동 수집
     · 순차 수집(for loop await)으로 API 과부하 방지


3. 구조 분석 (수정 없음)
   ────────────────────────
   - 계정별 데이터 분리 확인:
     Google Drive의 Index_Data 폴더가 각 계정 Drive에 개별 저장됨
     → 관리자가 업로드한 시장지표는 비관리자 계정에 공유되지 않음 (설계상 정상)
   - AdminPage.tsx는 사용자 승인 목록 관리 전용
     → 시장지표 업로드 기능 없음, 코드는 모든 계정에 동일하게 적용
